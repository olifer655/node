## session

session 是基于cookie的基础上，将真正的信息存储在服务上。

例如： 我们去理发店办理一张卡，并重置了100元钱，店员告诉了我们卡号。下一次我们再去理发店消费的时候，只需要告诉店员自己的卡号，店员在电脑上直接减去消费金额即可，顾客是无法修改卡内剩余金额数目的。

为了更加安全，服务器也可以再次对卡号进行加密处理。而在服务端存储的信息表就是session表，其余的都和cookie类似。
